# proj22-local-messager
### 项目名称
终端端侧通信子系统框架

### 项目描述

-   此项目的目的在于为基于linux的Android系统（其他Linux嵌入式系统也适用），提供一套方便的通信机制，不需要在系统中添加复杂的中间层接口，不需要受到系统权限的限制。

-   基于Android
    AOSP项目，在kernel中增加通信子系统框架，建立精简的HTTP本地服务，其他模块可以通过HTTP
    PUT发送数据到指定服务，也可以通过HTTP GET获取服务数据。

-   发送数据、接收数据的权限可以配置，收发请求时需要对请求进行权限校验审核。

-   发送数据、接收数据的格式可以根据不同服务定制。

-   收发数据需要有完整性校验；

-   该通信子系统的开销需要远小于传统HTTP
    Server，不能影响Android系统响应速度和功耗。

-   可以方便的扩展配置为局域网HTTP服务器。

-   项目可以基于源码：https://mirrors.tuna.tsinghua.edu.cn/help/AOSP/

### 注意：下面的内容是建议内容，不要求必须全部完成。选择本项目的同学也可与导师联系，提出自己的新想法，如导师认可，可加入预期目标

### 第一题：

-   方案可以以模块集成到Android项目。

-   被授权的模块，可以通过HTTP链接向该系统的指定服务发送数据，也可以通过HTTP向该系统的指定服务获取数据；此过程不需要增加额外的Android系统配置。

-   提供完整可以运行于Android的子系统代码和客户端示例代码。

-   该方案可以方便的扩展到局域网，例如当终端连接WIFI的时候，其他设备可以通过HTTP向该系统的指定服务收送数据。
